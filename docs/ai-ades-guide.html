용어집

AI-ADES 핵심 용어 해설

이 시스템에서 사용되는 AI·ML·공정 기술 용어를 쉽게 설명합니다.

Bayesian Optimization AI
베이지안 최적화
이전 실험 결과를 분석해서 다음 시도 방향을 계산하는 지능형 탐색 방법. 무작위로 시도하는 것보다 훨씬 적은 횟수로 최적값을 찾습니다.
AI-ADES 적용: OK 파라미터 구간(전체의 13.7%)을 수동으로 찾으면 30~50회 실험이 필요하지만, Bayesian Opt는 5~7회 만에 수렴합니다.
XGBoost ML
Extreme Gradient Boosting
수백 개의 결정 트리가 서로의 오류를 보완하면서 예측하는 앙상블 머신러닝 알고리즘. 표(엑셀) 형태 데이터에서 가장 검증된 알고리즘 중 하나입니다.
AI-ADES 적용: kerf_model(가공 폭 예측), depth_model(잔존 두께 예측), quality_model(OK/미가공/과가공 분류) 3개 모델로 운영합니다.
Optuna ML
하이퍼파라미터 자동 최적화 라이브러리
머신러닝 모델의 설정값(하이퍼파라미터)을 자동으로 찾아주는 도구. "트리를 몇 개 만들지, 얼마나 깊게 만들지"를 100회 자동 탐색합니다.
AI-ADES 적용: n_estimators(100~500), max_depth(3~8), learning_rate(0.01~0.3) 등을 각 모델마다 100회씩 자동 최적화합니다.
SHAP ML
SHapley Additive exPlanations
AI가 왜 그런 예측을 했는지 설명해주는 기법. "Power가 결과의 44%를 결정했다"처럼 각 변수의 기여도를 수치로 표현합니다.
AI-ADES 적용: 운영자가 AI 제안을 신뢰할 수 있도록 파라미터별 영향도를 막대 차트로 시각화해 승인 화면에 표시합니다.
n_estimators ML
트리 개수 하이퍼파라미터
XGBoost에서 만들 결정 트리의 개수. 너무 적으면 정확도가 낮고(과소적합), 너무 많으면 데이터를 통째로 외워버려 새 조건에 못 맞춥니다(과적합).
AI-ADES 탐색 범위: 100~500 사이에서 Optuna가 최적값을 자동 탐색합니다.
과적합 (Overfitting) ML
Overfitting
모델이 학습 데이터를 통째로 외워버려서 새로운 데이터에는 오히려 틀리는 현상. 시험 문제 정답을 암기했지만 응용 문제는 못 푸는 것과 같습니다.
AI-ADES 대응: K-Fold 교차검증(5회)과 Hold-out 20% 검증 세트로 과적합 여부를 지속 모니터링합니다.
R² (결정계수) ML
Coefficient of Determination
회귀 모델의 정확도를 0~1로 표현한 지표. 1에 가까울수록 예측이 실제값과 일치합니다. 0.75면 실제 변동의 75%를 모델이 설명한다는 의미입니다.
AI-ADES 목표: depth_model R² ≥ 0.75 (소규모·불균형 데이터 감안). 실설비 데이터 누적 시 0.90+ 목표.
F1 Score ML
분류 모델 성능 지표
분류 모델에서 정확도와 재현율의 조화평균. OK처럼 데이터가 적은 클래스를 얼마나 잘 맞추는지 측정하는 데 특히 유용합니다.
AI-ADES 목표: quality_model의 OK 클래스 F1 ≥ 0.60. OK가 328건 중 45건(13.7%)밖에 없어 클래스 가중치를 높여 학습합니다.
LSTM ML
Long Short-Term Memory (장단기 메모리)
시간의 흐름이 있는 데이터(시계열)를 학습하는 딥러닝 모델. 과거 패턴을 기억하면서 다음 상태를 예측하는 데 강합니다.
AI-ADES 적용(Phase 6): Plasma 센서의 250kHz 파형 데이터를 학습해 가공 이상을 실시간으로 감지합니다.
앙상블 (Ensemble) ML
Ensemble Learning
여러 모델의 예측을 합쳐 더 정확한 결과를 만드는 방법. 전문가 한 명보다 여러 전문가의 의견을 종합하면 더 정확한 것과 같은 원리입니다.
AI-ADES 장기 전략: XGBoost(표 데이터)와 LSTM(Plasma 파형)을 함께 사용하는 앙상블 구조로 진화합니다.
Human-in-the-Loop AI
인간 참여형 AI 의사결정
AI가 모든 것을 자동으로 하지 않고, 중요한 결정 단계에서 사람이 검토하고 승인하는 설계 방식. AI의 실수를 사람이 잡아주는 안전장치입니다.
AI-ADES 적용: AI가 파라미터를 제안하면 운영자가 [승인/거부]를 선택합니다. AI가 설비를 직접 제어하지 않습니다.
Auto DOE AI
Automated Design of Experiments
실험 조건을 AI가 자동으로 설계하고 순서를 결정하는 방법. 기존에 사람이 경험으로 정하던 실험 순서를 AI가 최적화합니다.
AI-ADES 적용: Bayesian Optimization 기반으로 다음 실험 조건을 자동 제안. 기존 30~50회 → 5~7회로 실험 횟수 87% 절감 목표.
LLM AI
Large Language Model (대형 언어 모델)
방대한 텍스트를 학습해서 사람처럼 글을 이해하고 생성하는 AI 모델. ChatGPT, Claude, Qwen 등이 대표적입니다.
AI-ADES 적용: SHAP 분석 수치를 "Power 15.2W에서 OK 확률이 87%입니다" 같은 자연어로 변환해 운영자에게 설명합니다.
Kerf 공정
가공 폭 (절단 선 너비)
레이저가 소재를 절단할 때 생기는 홈의 너비(μm). 레이저 에너지와 속도에 따라 달라집니다. kerf_model이 이 값을 예측합니다.
AI-ADES 데이터: 실험 후 현미경으로 계측. 예측값과 실제값의 오차가 작을수록 모델 정확도가 높습니다.
Depth (잔존 두께) 공정
Remaining Thickness
레이저 가공 후 Film에 남아있는 두께(μm). 이 값이 OK 범위(0~25μm)에 들어야 합니다. AI-ADES의 핵심 예측 대상입니다.
판정 기준: Depth > 25μm → 미가공 / 0 < Depth ≤ 25μm → OK / Depth = 0μm → 과가공
Defocus 공정
레이저 초점 이탈 거리
레이저 초점이 소재 표면에서 Z축 방향으로 얼마나 벗어났는지를 나타내는 거리(mm). 0이면 정확히 초점이 맞은 상태입니다.
AI-ADES 탐색: 0, 1, 2, 3, 4mm 중 최적값을 Auto DOE가 탐색합니다. SHAP 분석에서 Power 다음으로 중요한 변수로 예상됩니다.
Plasma Sensor 공정
플라즈마 센서 (Photodiode 기반)
레이저 가공 중 발생하는 플라즈마 빛을 감지하는 센서. 250kHz(초당 25만 번) 속도로 Plasma, Temperature, Reflection 신호를 기록합니다.
AI-ADES Phase 6: 이 센서의 시계열 파형을 LSTM이 학습해 가공 이상을 실시간 감지합니다. 신호가 없으면 출력 부족(7건에서 확인).
FastAPI 시스템
Python 고성능 웹 API 프레임워크
Python으로 API 서버를 빠르게 만들 수 있는 프레임워크. AI-ADES의 3개 Agent가 모두 FastAPI로 구현됩니다.
AI-ADES 적용: /predict, /doe/suggest, /doe/approve 등 엔드포인트를 통해 React 프론트엔드와 통신합니다.
Docker / Docker Compose 시스템
컨테이너 기반 가상화 플랫폼
각 서비스를 독립된 컨테이너로 격리해서 실행하는 기술. "내 PC에서는 됐는데 서버에서 안 된다"는 문제를 해결합니다.
AI-ADES 적용: 6개 서비스(Agent 3개, MLflow, InfluxDB, Grafana)를 docker-compose up 명령 하나로 일괄 기동합니다.
MLflow 시스템
머신러닝 실험 관리 플랫폼
AI 모델 학습의 모든 기록(파라미터, 성능, 모델 파일)을 버전별로 저장하고 비교하는 도구. 어떤 설정에서 최고 성능이 나왔는지 추적합니다.
AI-ADES 적용: Optuna 100회 탐색 결과와 최종 채택 모델을 http://localhost:5000 에서 확인합니다.
InfluxDB 시스템
시계열 전용 데이터베이스
시간 순서대로 쌓이는 데이터(센서값, 실험 결과 등)를 효율적으로 저장하고 조회하는 데이터베이스. 일반 DB보다 시계열 처리가 훨씬 빠릅니다.
AI-ADES 적용: 328건 실험 결과와 Plasma 센서 신호를 저장. Grafana가 이 데이터를 읽어 실시간 대시보드를 구성합니다.
PostgreSQL 시스템
오픈소스 관계형 데이터베이스
데이터를 표 형태로 저장하고 SQL로 조회하는 데이터베이스. AI-ADES의 실험 데이터, 레시피, 모델 메타데이터를 저장합니다.
AI-ADES 적용: experiments(328건), recipes, plasma_timeseries 3개 테이블. 기존 rag-pm-matcher와 공유하되 ades_db 스키마로 분리합니다.
Kafka 시스템
분산 메시지 스트리밍 플랫폼
대량의 데이터를 실시간으로 전달하는 메시지 버스. 설비에서 발생하는 센서 데이터를 끊김 없이 시스템으로 전달합니다.
AI-ADES 적용: Phase 6 실설비 연동 시 Plasma 센서 250kHz 데이터를 실시간으로 수집·전달하는 파이프라인에 사용됩니다.
Grafana 시스템
데이터 시각화 모니터링 플랫폼
InfluxDB, PostgreSQL 등 여러 DB의 데이터를 연결해 실시간 대시보드를 만드는 도구. 가공 품질 추이와 AI 성능을 한눈에 확인합니다.
AI-ADES 적용: http://localhost:3010 에서 품질 분포 파이 차트, Speed×Power 히트맵, DOE 수렴 그래프를 실시간 모니터링합니다.
Ollama 시스템
로컬 LLM 실행 플랫폼
인터넷 없이 로컬 PC에서 LLM을 실행할 수 있게 해주는 도구. 데이터를 외부로 보내지 않아 보안이 중요한 공장 환경에 적합합니다.
AI-ADES 적용: Qwen2.5:14b 모델을 기본으로 사용. 인터넷 단절 환경에서도 LLM 자연어 설명 기능이 동작합니다.
Gaussian Process ML
가우시안 프로세스
관측된 실험 결과로부터 아직 시도하지 않은 구간의 결과를 확률적으로 추정하는 수학적 모델. Bayesian Optimization의 핵심 엔진입니다.
AI-ADES 적용: BoTorch 라이브러리로 구현. 실험할 때마다 업데이트되어 점점 더 정확한 탐색 지도를 그립니다.
K-Fold 교차검증 ML
K-Fold Cross Validation
데이터를 K등분해서 한 구간씩 번갈아 테스트 세트로 사용하며 모델 성능을 평가하는 방법. 데이터가 적을 때 신뢰할 수 있는 성능 측정에 사용합니다.
AI-ADES 적용: 328건 데이터를 5등분(k=5)해서 성능 측정. 매번 다른 65.6건이 테스트 세트가 되어 과적합 여부를 검증합니다.
디지털 트윈 AI
Digital Twin
실제 설비와 공정을 컴퓨터 안에 똑같이 구현한 가상 모델. 실제 실험 없이 가상으로 결과를 시뮬레이션할 수 있어 실험 비용을 0으로 만드는 것이 목표입니다.
AI-ADES 장기 목표: 충분한 실험 데이터 축적 후 디지털 트윈 구축. 신규 소재 투입 시 실제 레이저 없이 PC에서만 최적 파라미터를 결정합니다.
Energy Density 공정
에너지 밀도 (파생 변수)
Power(출력) ÷ Speed(속도)로 계산하는 단위면적당 레이저 에너지량(W·s/mm). 속도가 느릴수록, 출력이 높을수록 에너지 밀도가 높아져 깊이 가공됩니다.
AI-ADES 적용: Feature Engineering에서 파생 변수로 추가. Power와 Speed 단독보다 이 조합이 Depth를 더 잘 예측하는 경우가 많습니다.
