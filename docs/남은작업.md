## A. 오늘 작업의 후속 (바로 가능 · 선택)

| # | 작업 | 내용 | 비고 |
|---|---|---|---|
| A-1 | **AI 채팅 LLM도 Ollama로 전환** | 현재 AI 설명은 Ollama(nb-ai-ws GPU)로 전환됐으나 **AI 채팅(`CHAT_LLM_PROVIDER`)은 아직 Claude**(토큰 과금). `.env`에서 `CHAT_LLM_PROVIDER=ollama`로 바꾸고 execution-agent 재생성하면 채팅도 nb-ai-ws GPU 사용(비용 0·폐쇄망). | 채팅은 자유 대화라 14B 로컬 품질이 Claude보다 떨어질 수 있어 보류해둔 상태. 전환 여부는 품질 vs 비용 판단 |
| A-2 | nb-ai-ws 미사용 스택 정리 | nb-ai-ws가 안 쓰는 ades 스택(data-prep·modeling·postgres·influxdb·mlflow·grafana·kafka)을 한 벌 더 돌리는 중. LLM-only 오프로드에선 불필요 → GPU 박스를 가볍게 두려면 **Ollama만 남기고** 나머지 내려도 됨 | 선택. 나중에 전체 마이그레이션(A-3) 시 다시 사용 |
| A-3 | (먼 훗날) 데이터 전체 마이그레이션 | 데이터/DB/모델까지 nb-ai-ws로 옮겨 단일 백엔드화. 로컬 Postgres(실험 350건 등)·InfluxDB·MLflow·모델을 덤프·복원 후 `.env.nb-ai-ws` 프로필로 전환 | 현재는 데이터가 Surface 로컬에만 있어 보류. 필요해질 때 이관 스크립트 작성 |
