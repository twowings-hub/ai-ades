오늘 한 일

고객사 미팅 전, 미리 준비할 수 있는
3가지를 먼저 만들었습니다

설비와 직접 연결되는 기능은 고객사 설비 사양(통신 방식, 측정값 종류 등)을 알아야 만들 수 있습니다. 하지만 "데이터를 어떻게 처리할지", "AI가 어떻게 학습할지"는 미리 골격을 만들어 둘 수 있습니다. 아래 3가지가 바로 그 작업입니다.

1
데이터 수집 — 입구 만들기
Plasma 센서 데이터를 시스템에 넣는 통로 (Plasma ZIP 파서)
①왜 필요한가

레이저로 가공할 때 Plasma 센서가 1초에 25만 번(250kHz) 빛의 세기·온도 등을 측정합니다. 이 신호 안에는 "가공이 잘 되고 있는지 / 이상이 생겼는지"를 알려주는 단서가 들어 있습니다. 그런데 지금까지는 이 데이터를 시스템 DB에 넣을 방법 자체가 없었습니다. 앞으로 만들 AI 이상감지 모델(2번 항목)을 학습시키려면 먼저 이 데이터가 DB에 들어와야 합니다.

②어떻게 구현했는가

센서 장비가 만들어내는 ZIP 파일(실험번호별 CSV 묶음)을 업로드하면, 시스템이 자동으로 압축을 풀고 각 CSV(세미콜론으로 구분된 7개 측정값 컬럼)를 읽어 "실험번호 + 시간 + 측정값" 형태의 표로 변환합니다. 그 후 기존 실험 기록과 실험번호를 대조해서 어느 실험의 데이터인지 매칭하고, DB에 대량으로 저장합니다.

FastAPI POST /data/plasma/upload pandas (ZIP/CSV 파싱) PostgreSQL plasma_timeseries
검증 결과 (합성 테스트 데이터) ──
업로드 파일 : AI-ADES_test_plasma_sample.zip
포함 실험 : 5개 실험 (exp_no 1~5), 각 500개 측정 시점
매칭 결과 : 5개 실험 모두 매칭, 정상 적재
────────────────────────────
→ 검증 후 테스트 데이터는 삭제하여 DB를 원상태로 유지
고객사 확인 필요 — ① ZIP 안의 CSV 파일명이 실험번호와 동일한지, ② 시간 컬럼의 단위가 초(second)가 맞는지, ③ 데이터의 실제 측정 시각을 어떻게 기록할지 (현재는 업로드 시각을 기준으로 임시 계산 중).
2
AI 모델 — 이상 신호를 스스로 찾는 눈
레이저 파형에서 "평소와 다른 패턴"을 찾는 AI (LSTM-Autoencoder)
①왜 필요한가

지금까지 AI(XGBoost)는 Speed·Power 같은 "숫자로 정해진 조건"만 보고 결과를 예측했습니다. 하지만 같은 조건이라도 레이저가 쏘는 동안 미세한 떨림, 먼지, 초점 흔들림 등으로 Plasma 파형 자체가 평소와 달라지는 경우가 있습니다. 이런 미세한 이상은 사람이 파형을 일일이 들여다봐도 찾기 어렵고, "정상/비정상"이라고 라벨을 붙여줄 사람도 없습니다. 그래서 "정상적인 파형 패턴만 보여주고, 거기서 벗어나면 이상으로 판단"하는 AI(비지도학습)가 필요합니다.

②어떻게 구현했는가

LSTM-Autoencoder라는 구조를 사용합니다. 동작 원리는 간단합니다 — 긴 파형을 짧게 압축했다가(인코더) 다시 원래대로 복원합니다(디코더). 정상 파형은 압축→복원해도 원본과 거의 똑같지만, 이상한 파형은 복원이 잘 안 되어 "복원 오차"가 커집니다. 정상 데이터로 학습한 뒤, 이 오차가 평소(정상 데이터)의 상위 5% 수준을 넘으면 "이상 신호"로 판정합니다. 파형은 50개 시점 단위로 잘라(슬라이딩 윈도우) 학습에 사용합니다.

PyTorch (LSTM-Autoencoder) Sliding Window (50개 단위) MLflow 모델 기록 POST /model/train-anomaly POST /predict-anomaly
검증 결과 (합성 테스트 데이터) ──
PyTorch 포함 이미지 재빌드 → 학습/예측 정상 동작 확인
이상 판정 기준(threshold) : 정상 복원오차의 상위 5%
예측 결과 : 실험을 M1/M2/Blank 3구간으로 나누어 구간별 이상비율 산출
────────────────────────────
→ 모델/스케일러/기준값을 저장, MLflow에 실험 기록
고객사/실데이터 확인 필요 — 현재 M1/M2/Blank 구간 구분은 "데이터를 3등분"하는 임시 방식입니다. 실제로는 가공 단계가 바뀌는 시점을 파형에서 자동으로 찾아내는 로직으로 교체해야 합니다. 또한 윈도우 크기 등 설정값은 실제 현장 데이터로 다시 조정이 필요합니다.
3
운영 자동화 — AI가 스스로 다시 공부한다
실험 데이터가 쌓이면 자동으로 재학습하는 구조 (경량 자동 재학습 트리거)
①왜 필요한가

AI 모델은 데이터가 쌓일수록 더 똑똑해집니다. 하지만 매번 운영자가 "이제 데이터가 많이 쌓였으니 재학습해야지"를 직접 판단해서 버튼을 누르는 건 번거롭고 잊어버리기 쉽습니다. 그렇다고 별도의 자동화 서버(Airflow 등)를 새로 띄우면 개발용 PC(Surface, 32GB 메모리)에 부담이 됩니다.

②어떻게 구현했는가

새로운 별도 서버 없이, 기존 시스템 안에 가벼운 점검 로직을 추가했습니다. 새 실험 결과가 하나 저장될 때마다(Excel 업로드, Auto DOE 결과 입력 등) "현재까지 누적된 전체 실험 건수"와 "마지막으로 모델을 학습했을 때의 건수"를 비교합니다. 그 차이가 50건(설정 가능) 이상이면, 시스템이 알아서 백그라운드로 재학습을 시작하고, 누가 언제 왜 재학습했는지 기록(감사로그)을 남깁니다.

FastAPI 백그라운드 스레드 .env AUTO_RETRAIN_THRESHOLD=50 audit_logs 자동 기록 POST /admin/model/check-auto-retrain
검증 결과 (실제 운영 데이터) ──
누적 실험 건수 : 338건 / 마지막 학습 시점 : 328건
→ 차이가 기준치를 넘어 실제 자동 재학습 실행됨
재학습 결과 : Depth 모델 R²=0.83, 품질 모델 F1(OK)=0.72
────────────────────────────
→ 실제 운영 모델이 최신 데이터로 갱신된 정식 결과 (테스트용 더미 아님)

핵심 용어 해설

오늘 작업에서 사용된 AI·데이터·운영 기술 용어를 쉽게 풀어 설명합니다.

LSTM AI
Long Short-Term Memory
시간 순서가 있는 데이터(파형, 문장 등)를 분석할 때, "바로 직전 값"뿐 아니라 "한참 전의 흐름"까지 기억하면서 패턴을 파악하는 AI 신경망 구조입니다.
AI-ADES 적용: Plasma 센서가 250kHz로 수집한 시계열 신호의 흐름(파형 모양)을 학습하는 데 사용합니다.
Autoencoder AI
오토인코더 (자기 복원 신경망)
입력 데이터를 일부러 작게 압축했다가 다시 원래 크기로 복원하도록 학습시키는 AI입니다. "정상 데이터는 잘 복원되고, 처음 보는 이상한 데이터는 복원이 잘 안 된다"는 성질을 이용합니다.
AI-ADES 적용: LSTM과 결합한 LSTM-Autoencoder로 Plasma 파형의 정상 패턴을 학습하고, 복원이 잘 안 되는 구간을 "이상 신호"로 표시합니다.
비지도학습 AI
Unsupervised Learning
"정답(라벨)"이 없는 데이터로 학습하는 방식입니다. 사람이 일일이 "이건 정상, 이건 이상"이라고 표시해주지 않아도, 데이터 안의 패턴 자체를 스스로 익힙니다.
AI-ADES 적용: Plasma 파형에는 "이상 여부" 라벨이 없으므로, 정상 파형만으로 학습시켜 패턴에서 벗어나는 것을 자동으로 찾아냅니다.
복원 오차 (Reconstruction Error) AI
Reconstruction Error
Autoencoder가 압축했다가 복원한 결과와 원본 데이터의 차이입니다. 이 차이가 클수록 "AI가 처음 보는 낯선 패턴"이라는 뜻입니다.
AI-ADES 적용: 정상 데이터의 복원 오차 분포 중 상위 5% 지점을 "이상 판정 기준선(threshold)"으로 사용합니다.
슬라이딩 윈도우 (Sliding Window) AI
Sliding Window
길게 이어진 시계열 데이터를 일정한 길이(예: 50개 시점)로 잘라서 여러 개의 작은 조각(샘플)을 만드는 방법입니다. 조금씩 겹치게 잘라서 데이터 양을 늘릴 수 있습니다.
AI-ADES 적용: 50개 시점 단위로 자르고, 10개씩 이동하며 잘라서 LSTM-Autoencoder 학습용 샘플을 생성합니다.
Plasma 센서 센서
Plasma Sensor
레이저가 소재를 가공할 때 발생하는 빛(플라즈마)의 세기, 온도, 반사율 등을 매우 빠른 속도(250kHz = 1초에 25만 번)로 측정하는 센서입니다.
AI-ADES 적용: Area, Plasma, P-Raw, Temp, T-Raw, Refl, R-Raw 7개 채널을 측정하며, 이 신호의 패턴 변화로 가공 이상을 조기에 감지하려 합니다.
시계열 데이터 (Time-series) 데이터
Time-series Data
시간 순서대로 기록된 데이터입니다. "몇 시 몇 분 몇 초에 어떤 값이었다"가 연속적으로 쌓인 형태로, 변화의 흐름(추세, 진동, 이상 패턴)을 분석하는 데 사용됩니다.
AI-ADES 적용: Plasma 센서가 만들어내는 250kHz 신호가 대표적인 시계열 데이터이며, plasma_timeseries 테이블에 저장됩니다.
Long format / Wide format 데이터
데이터 표 형식
같은 데이터를 표로 표현하는 두 가지 방식입니다. Long format은 "시간, 채널명, 값"처럼 한 줄에 하나의 측정값만 적는 방식이고 (DB 저장에 유리), Wide format은 채널마다 별도의 열(컬럼)을 만들어 한 시점의 모든 채널 값을 한 줄에 나열하는 방식입니다 (AI 모델 입력에 유리).
AI-ADES 적용: DB에는 long format으로 저장하고, LSTM 학습 시에는 wide format으로 변환해서 사용합니다.
자동 재학습 트리거 자동화
Auto-retrain Trigger
새로운 데이터가 일정량 이상 쌓이면, 사람이 시키지 않아도 AI 모델을 자동으로 다시 학습시키는 장치입니다.
AI-ADES 적용: 누적 실험 건수가 마지막 학습 시점보다 50건 이상 늘어나면 자동으로 재학습을 시작합니다 (.env에서 기준치 조정 가능).
감사로그 (Audit Log) 자동화
Audit Log
시스템에서 일어난 중요한 동작(승인, 거부, 설정 변경, 재학습 등)을 "누가, 언제, 무엇을, 왜" 했는지 기록해두는 이력입니다.
AI-ADES 적용: 자동 재학습이 실행되면 "system(auto)"가 실행했다는 기록과 그 사유(누적 건수)가 audit_logs 테이블에 자동 기록됩니다.
MLflow 자동화
머신러닝 실험 관리 플랫폼
AI 모델을 학습할 때마다 어떤 설정으로, 어떤 성능이 나왔는지를 자동으로 기록하고 비교할 수 있게 해주는 도구입니다.
AI-ADES 적용: 기존 XGBoost 모델뿐 아니라 이번에 추가한 LSTM-Autoencoder(PyTorch) 모델도 별도 방식으로 기록되도록 확장했습니다.
OPC-UA 현장연동
Open Platform Communications - Unified Architecture
공장의 제조 설비와 컴퓨터 시스템이 서로 데이터를 주고받을 때 널리 쓰이는 산업용 표준 통신 규격입니다. 설비의 현재 상태나 설정값을 표준화된 방식으로 읽고 쓸 수 있습니다.
AI-ADES 적용 예정: 고객사 CO₂ 레이저 설비가 OPC-UA를 지원하는 경우, 이 규격을 통해 파라미터 전송과 결과 수신을 자동화할 수 있습니다 (지원 여부는 고객사 확인 필요).
설비 통신 드라이버 현장연동
Equipment Communication Driver
AI-ADES 시스템과 실제 레이저 설비 사이에서 "통역" 역할을 하는 소프트웨어 모듈입니다. 설비마다 통신 방식(OPC-UA, RS-232 시리얼, Ethernet 소켓 등)이 다르기 때문에, 이를 표준화된 형태로 변환해주는 추상화 계층이 필요합니다.
AI-ADES 적용 예정: 고객사 설비의 통신 방식을 확인한 뒤, "AI 제안 파라미터 전송"과 "가공 결과 자동 수신" 기능을 이 드라이버를 통해 구현할 예정입니다.
